# Contributing

## Scope

- Treat `freshtrack-v2` as the active app.
- Root `src/` is legacy and should only be touched intentionally.

## Development workflow

1. Make changes in small, reviewable slices.
2. Prefer shared components over repeated route-level styling.
3. Keep route files thin; move feature logic to feature modules/hooks.
4. Ensure server mutations are auth-protected and household-scoped.

## Verification

Run before opening PR:

```bash
npm run typecheck
npm run build
npm run test:e2e
```

## Code organization guidelines

- `src/components/ui`: reusable primitives only.
- `src/components/layout`: shared shell/layout UI.
- `src/features/*`: feature-local components, hooks, and types.
- `src/routes/*`: route composition only.
- `src/server/*`: server functions and domain data access.

## Generated files

- `src/routeTree.gen.ts` is generated by TanStack Router.
- SQL in `drizzle/` is generated by Drizzle and should be committed with schema changes.
